# Key Quotes, Data Points, and Slide-Worthy Moments

## Core thesis phrases

- "The cost of code has essentially gone to near zero... the expensive part now is knowing what to build."
- "About 1% of people are using AI to actually build things... the other 99% are still just using ChatGPT for emails."
- "Developers are becoming orchestrators... you need product thinking, not just coding skills."
- "The real unlock isn't the AI model, it's the context you give it."
- "A lot of SaaS is about to discover their pricing assumed a world where building was hard."
- "Software malleability didn’t matter when the average user couldn’t change it. Now that the average person can prompt their features into existence, the platforms that allow that are going to be the winners."
- Old SaaS moat: 1,000 features, each customer uses 50, everyone needs a different 50, so nobody can leave.

## Hard numbers

- MarTech landscape: roughly 200 tools in 2011 → 15,000+ tools by 2024/2026 framing.
- DishList case: 837 restaurants exposed, 564 user ratings exposed, 9 tables readable, 3 tables writable by any authenticated user, full dump in about 3 minutes.
- Andrea case: Modal/GitHub Actions replaced by a custom scheduler in 3 hours; job runtime improved from ~40 min to ~10 min; translation quality evaluation scored 94.5/100 across 5 languages.
- Theo / Frame.io case: Frame.io acquired for $1.3B; majority functionality rebuilt part-time in two weeks with AI coding tools.
- Block layoffs example: ~4,000 employees / nearly half of Block, framed as a signal of changing software economics.
- T3 Code example: ~42–45k installs/opens, ~16k weekly users, ~9k stars, ~1.5k forks — evidence of users customizing software directly.
- Ghostty / libghostty example: main app reportedly reached ~1M daily users in 18 months; reusable building block reportedly surpassed it with multiple millions of daily users in ~2 months.

## Visual moments

1. **MarTech explosion chart** — start with scale: 200 → 15,000+.
2. **DishList contrast** — beautiful product screenshot vs raw database/API response.
3. **Software pipeline funnel** — old bottleneck at "write code" vs new bottleneck at review/test/verify/ship.
4. **Frame.io vs Lawn** — $1.3B acquisition vs 2-week part-time rebuild.
5. **Andrea Street Coding story** — frustration → AI debugging → 3 hours → new product.
6. **Old playbook vs new playbook** — SaaS UI/retention/seats vs skill/pattern/invocations.
7. **Old SaaS moat vs AI-native infrastructure** — 1,000 bundled features vs core primitives + API/CLI + agents building the weird 1%.

## Tensions to develop

- Cheap building creates both empowerment and risk.
- SaaS is not all dead: simple tools get repriced, infrastructure grows, trusted/critical platforms get stronger.
- AI does not remove human value; it moves value toward requirements, judgment, context, QA, security, and agency.
- Enterprise safety matters, but guardrailing without adoption becomes paralysis.
